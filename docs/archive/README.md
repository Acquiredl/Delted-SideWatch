# Archived Design Documents

This directory contains **historical architecture and planning documents** from
SideWatch's design phase. They are preserved for transparency — not because
they reflect the current state of the project.

## Why These Are Here

These documents were written during the planning phase as design proposals.
Some features described were subsequently built (with modifications); others
remain unimplemented. Rather than deleting them (which could look like we're
hiding something), we've moved them here with clear status annotations.

## How to Read These

Each document has an `## Archive Status` section at the top that explains:

- Whether the feature was implemented, partially implemented, or still planned
- Known factual errors in the original document
- What to look at instead for current truth (code, CLAUDE.md, other docs)

## Current Truth

For the actual state of the project, refer to:

- **[CLAUDE.md](../../CLAUDE.md)** — comprehensive project context, architecture, and implementation status
- **[README.md](../../README.md)** — user-facing feature summary
- **[docs/SECURITY.md](../SECURITY.md)** — security model and threat analysis
- **Code itself** — `services/manager/`, `services/gateway/`, `frontend/`
